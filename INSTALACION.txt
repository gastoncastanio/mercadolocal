⚙️ PASO 1: CONFIGURAR EL BACKEND

  cd backend

  # Instalar dependencias
  npm install

  # Crear archivo .env
  cat > .env << 'ENVFILE'
MONGODB_URI=mongodb+srv://usuario:contraseña@cluster.mongodb.net/logoai
OPENAI_API_KEY=sk-tu-clave-openai
PORT=3001
NODE_ENV=development
ENVFILE

  # IMPORTANTE: Edita el archivo .env con:
  # - Tu MongoDB URI (https://www.mongodb.com/cloud/atlas)
  # - Tu OpenAI API Key (https://platform.openai.com/account/api-keys)

═══════════════════════════════════════════════════════════════════════════════

⚙️ PASO 2: CONFIGURAR EL FRONTEND

  cd ../frontend

  # Instalar dependencias
  npm install

  # Crear archivo .env.local (opcional, usa defaults)
  cat > .env.local << 'ENVFILE'
VITE_API_URL=http://localhost:3001/api
ENVFILE

🚀 PASO 3: EJECUTAR LA APLICACIÓN

TERMINAL 1 - BACKEND:

  npm run dev
  
  ✅ Esperarás: "🚀 Servidor LogoAI escuchando en http://localhost:3001"

TERMINAL 2 - FRONTEND:

  npm run dev
  
  ✅ Esperarás: "VITE v5.0.0  ready in XXX ms"

❌ SI ALGO NO FUNCIONA:

Error: "OPENAI_API_KEY not found"
  → Verifica que backend/.env existe
  → Revisa tu API key en platform.openai.com

Error: "MongoDB connection failed"
  → Verifica MONGODB_URI en .env
  → Comprueba IP whitelist en MongoDB Atlas

Error: "CORS error"
  → Asegúrate que backend está en puerto 3001
  → Frontend debe acceder a http://localhost:3001/api

Error: "Port already in use"
  → Cambia PORT en backend/.env (default: 3001)
  → Cambia puerto Vite: npx vite --port 5174
